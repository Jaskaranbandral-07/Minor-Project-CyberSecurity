    Hash Algorithms

    Understanding how passwords are stored and why some are weaker

    Weak

    MD5

    128-bit hash, extremely fast to compute. Vulnerable to rainbow tables and collision attacks.

    Speed:~25 billion/sec (GPU)
    Hashcat Mode:-m 0
    password → 5f4dcc3b5aa765d61d8327deb882cf99
    Weak

    NTLM

    Windows password hash. No salting, fast to compute. Used in Active Directory environments.

    Speed:~77 billion/sec (GPU)
    Hashcat Mode:-m 1000
    password → a4f49c406510bdcab6824ee7c30fd852
    Moderate

    SHA-256

    256-bit hash, part of SHA-2 family. Better than MD5 but still too fast without salting.

    Speed:~8 billion/sec (GPU)
    Hashcat Mode:-m 1400
    password → 5e884898da2805...(64 chars)
    Strong

    bcrypt

    Adaptive hash with built-in salt and configurable work factor. Industry standard for passwords.

    Speed:~28,000/sec (GPU)
    Hashcat Mode:-m 3200
    password → $2b$12$LJ3m4y...

    🔑 Key Takeaway

    Fast hashes like MD5 and NTLM are not designed for password storage. Modern systems should use bcrypt, scrypt, or Argon2 which are intentionally slow to resist brute-force attacks.

    Salting & Defense

    How salting and proper hashing protect against cracking attacks

    What is Salting?

    Without Salt
    "password" → MD5() → 5f4dcc3b...

    Same input = Same hash. Vulnerable to rainbow tables.

    With Salt
    "password" + "x9$kQ2" → MD5() → a8f5e103...

    Unique salt per user = Unique hash. Rainbow tables useless.

    🧂

    Use Unique Salts

    Generate a random salt for each password. Makes precomputed attacks infeasible.

    🐢

    Use Slow Hashing

    bcrypt, scrypt, Argon2 are designed to be slow. Each guess costs significant computation.

    📏

    Enforce Complexity

    Minimum 12 characters, mix of upper/lower/digits/symbols. Block common passwords.

    🔐

    Multi-Factor Auth

    Even if cracked, MFA adds a second barrier. Use TOTP, hardware keys, or biometrics.

    Hashcat Commands Reference

    Essential commands for real-world Hashcat usage

    Dictionary Attack

    hashcat -m 1000 -a 0 hashes.txt rockyou.txt -o cracked.txt

    Rule-Based Attack (best64)

    hashcat -m 1000 -a 0 hashes.txt rockyou.txt -r rules/best64.rule -o cracked.txt

    Brute Force (8 chars, mixed)

    hashcat -m 1000 -a 3 hashes.txt ?a?a?a?a?a?a?a?a -o cracked.txt

    Show Cracked Results

    hashcat -m 1000 hashes.txt --show

    Benchmark GPU Performance

    hashcat -b -m 1000
